# 山东行程调整工具 · 分享使用教程

本教程面向**不懂代码**的普通使用者。页面运行时只有一个 HTML 文件，但腾讯 JSAPI GL 不支持 `file://`；要稳定使用完整地图功能，应通过静态托管网址或本地 HTTP 服务访问，并保持联网。

---

## 一、静态网站分享（推荐）

### 1. 分享初始行程

1. 将单个 **`itinerary.html`** 重命名为 **`index.html`**。
2. 上传到任意静态托管平台，例如 [GitHub Pages](https://pages.github.com/)、[Cloudflare Pages](https://pages.cloudflare.com/) 或 [Netlify](https://www.netlify.com/)。
3. 按平台说明完成部署并取得 HTTPS 网址。
4. 将这个网址分享给接收者。接收者通过网址打开后，才能稳定使用腾讯地图、地点搜索和地图选点。

### 2. 分享你调整后的行程

若你在页面上新增地点、编辑地点或调整时间后，请按以下步骤：

1. 在页面顶部找到 **「导出 JSON」** 按钮并点击。
2. 浏览器会下载一个文件，文件名类似：`shandong-itinerary-2026-07-16.json`（日期以实际导出日为准）。
3. 将刚导出的 `shandong-itinerary-YYYY-MM-DD.json` 发给接收者，并同时提供上面的静态托管网址。

### 3. 接收者如何导入

1. 通过分享者提供的静态托管网址打开页面（需联网）。
2. 点击 **「导入 JSON」**。
3. 在文件选择框中选中对方发来的 `.json` 文件。
4. 在确认对话框中点击 **确定**。
5. 页面会显示与导出时一致的行程。

> **注意：** 静态托管不会让不同访问者自动同步修改；调整后的行程仍需通过导出和导入 JSON 交换。

## 二、保存边界（请务必阅读）

- 你新增的地点和调整后的顺序，**只保存在当前浏览器和当前网址来源下**。
- **不同人的数据不会自动同步。** 每人各自新增地点或调整顺序，互不影响。
- 在以下情况发生**之前**，务必先 **导出 JSON** 备份：
  - 换浏览器（如从 Chrome 换到 Edge）
  - 换电脑或手机
  - 清理浏览器数据 / 清除网站数据
  - 使用无痕 / 隐私模式
  - 改变访问地址（例如从本地 HTTP 地址改为静态托管网址）
- **JSON 是推荐的备份方式。** 每次新增地点或调整顺序后若要再次分享，需重新导出最新 JSON。

## 三、本地测试

不要通过双击 `itinerary.html` 测试完整地图功能。请在项目目录打开 PowerShell，运行：

```
C:/Python314/python.exe -m http.server 8000
```

然后在浏览器访问：

```
http://localhost:8000/itinerary.html
```

直接双击会使用 `file://`。这种方式下地图、地点搜索和地图选点**不可保证可用**；DAY 时间轴、导入、导出、恢复和部分本地保存功能可能仍可用。页面会明确显示地图不可用提示。

## 四、腾讯地图 Key（正式分享前必读）

当前页面内嵌的是**有有效期的临时 Key**，适合开发测试，**不适合长期对外分享**。

### 正式分享前请完成

1. 前往 [腾讯位置服务](https://lbs.qq.com/) 申请 **JavaScript API GL** 的长期 Key。
2. 在控制台开启所需能力：
   - **JSAPI**（地图显示、标记、路线等）
   - **WebService / service**（地点搜索建议）
3. 若使用静态网站分享，在 Key 的**域名白名单**中加入你的托管域名（如 `你的用户名.github.io`）。
4. 将新 Key 交给**维护页面的人**（通常是有 Python 环境的开发者），在 `generate_html.py` 文件**顶部**找到 `KEY = "..."` 一行，替换为新 Key，然后运行：

   ```
   C:/Python314/python.exe generate_html.py
   ```

5. 重新生成后，用新的 `itinerary.html` 进行分享。

> **安全提示：** Key 会出现在浏览器页面源码中，任何人可见。这是前端地图的正常做法，**不要把 Key 当作服务器密码**；请用域名白名单和配额限制来管控滥用。

**普通接收者无需安装 Python**，只需访问部署了新 Key 页面的网址。

## 五、页面基本操作速查

| 操作 | 方法 |
|------|------|
| 切换日期 | 点击顶部 DAY 1～DAY 9 标签 |
| 新增地点 | 点击「添加点」，在弹出的表单中填写信息；可搜索地点，或先点击「地图选点」按钮再单击地图，最后提交 |

| 备份调整结果 | 点击「导出 JSON」 |
| 恢复初始 | 点击「恢复初始行程」 |
| 导入他人行程 | 点击「导入 JSON」并选择文件 |

## 六、常见问题

### 地图空白

可能原因与处理：

1. **未联网** — 连接网络后刷新页面。
2. **Key 已过期** — 联系分享者更新 Key 并重新生成 `itinerary.html`。
3. **域名未授权** — 若通过网站打开，需在腾讯控制台把该域名加入 Key 白名单。
4. **浏览器拦截脚本** — 尝试换用 Chrome / Edge，并允许加载外部脚本。
5. **直接双击 HTML** — 腾讯 JSAPI GL 不支持 `file://`；请按「本地测试」启动 HTTP 服务，或改用静态托管网址。

### 新增地点或排序结果消失了

1. 若有 JSON 备份：点击「导入 JSON」，选择最近导出的文件。
2. 若无备份：只能点击「恢复初始行程」回到默认行程，之前新增的地点或调整后的顺序无法找回。

### 地点搜索不可用

1. 检查 Key 是否开启 **WebService / service** 权限。
2. 若搜索仍失败，可改用 **地图选点**：点击「添加点」打开表单，点击「地图选点」按钮，再在地图上单击目标位置，坐标会自动填入表单。

### 顺序没有按预期变化

- 左侧行程和地图都按**时间**自动排序，不是手动拖拽顺序。
- 如需把某个点提前，请编辑该点并把时间改早一些。

### 导入 JSON 后没有变化

- 确认选择的是正确的 `.json` 文件（导出时的 `shandong-itinerary-YYYY-MM-DD.json`）。
- 若 JSON 文件损坏或格式不对，页面会提示错误，**不会**覆盖当前行程。

## 七、验收清单（收到最终文件后请逐项勾选）

请在**联网**的**桌面浏览器**中通过静态托管网址打开页面，或按「本地测试」访问 `http://localhost:8000/itinerary.html`，逐项测试并勾选：

- [ ] **DAY 切换：** 依次点击 DAY 1、DAY 2、DAY 3，地图与时间轴均切换到对应日期，标记带编号，路线带箭头。
- [ ] **地图缩放：** 放大、缩小地图时，标记位置固定、不聚合为簇。
- [ ] **地图选点新增：** 点击「添加点」打开弹出表单，点击「地图选点」按钮，再单击地图填入坐标；提交后新点出现在列表和地图上，刷新页面后仍存在。
- [ ] **搜索新增：** 点击「添加点」打开弹出表单，在搜索框输入地名并选择建议项；提交后新增成功，刷新后仍存在。
- [ ] **编辑已有地点：** 点击某张卡片的「编辑」，修改标题或时间后保存，左侧顺序和地图编号同步更新。
- [ ] **按时间排序：** 新增或编辑后，当天行程按时间从早到晚排列，地图路线顺序一致。
- [ ] **导出 → 恢复初始 → 导入：** 先导出 JSON；点击「恢复初始行程」回到默认行程；再导入刚才的 JSON，行程完整恢复。
- [ ] **损坏 JSON：** 用文本编辑器故意破坏 JSON 内容后尝试导入，页面应报错且**不改变**当前行程。

若某项无法完成，请记录浏览器名称、实际访问 URL 和具体现象，反馈给分享者。
