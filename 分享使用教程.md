# 山东行程编辑器 · 分享使用教程

本教程面向**不懂代码**的普通使用者。你只需一个网页文件（`itinerary.html`）即可查看和编辑行程；需要联网加载腾讯地图。

---

## 一、直接发送文件（最简单）

### 1. 分享初始行程

只发送 **`itinerary.html`** 这一个文件即可。

接收者双击打开（或在浏览器中选择“打开文件”），即可查看完整 9 天行程和地图。

### 2. 分享你修改后的行程

若你在页面上新增、删除、排序或编辑过地点，请按以下步骤：

1. 在页面顶部或侧边栏找到 **「导出 JSON」** 按钮并点击。
2. 浏览器会下载一个文件，文件名类似：`shandong-itinerary-2026-07-16.json`（日期以实际导出日为准）。
3. 将 **两个文件一起发送** 给接收者：
   - `itinerary.html`
   - 刚导出的 `shandong-itinerary-YYYY-MM-DD.json`

### 3. 接收者如何导入

1. 双击打开 `itinerary.html`（需联网）。
2. 点击 **「导入 JSON」**。
3. 在文件选择框中选中对方发来的 `.json` 文件。
4. 在确认对话框中点击 **确定**。
5. 页面会显示与导出时一致的行程。

> **注意：** 打开页面时需要联网，以便加载腾讯地图。若地图区域空白，请参阅下文「常见问题」。

## 二、保存边界（请务必阅读）

- 你在页面上做的修改，**只保存在当前浏览器、当前打开方式下**（例如 Chrome 直接打开本地文件）。
- **不同人的数据不会自动同步。** 每人各自编辑，互不影响。
- 在以下情况发生**之前**，务必先 **导出 JSON** 备份：
  - 换浏览器（如从 Chrome 换到 Edge）
  - 换电脑或手机
  - 清理浏览器数据 / 清除网站数据
  - 使用无痕 / 隐私模式
  - 改变打开方式（例如从「双击本地文件」改为「通过网站链接打开」）
- **JSON 是推荐的备份方式。** 每次修改后若要再次分享，需重新导出最新 JSON。

## 三、静态网站分享（可选）

若希望他人通过网址访问（而非收发文件），可将页面放到任意免费静态托管服务上。

### 操作步骤

1. 将 `itinerary.html` **重命名**为 `index.html`。
2. 上传到静态托管平台，例如（任选其一即可，不必绑定特定平台）：
   - [GitHub Pages](https://pages.github.com/)
   - [Cloudflare Pages](https://pages.cloudflare.com/)
   - [Netlify](https://www.netlify.com/)
3. 按平台说明完成部署，获得一个网址（如 `https://你的用户名.github.io/项目名/`）。
4. 把网址发给他人即可访问。

### 重要说明

- 静态托管**同样不会**让不同访问者自动同步修改；每人仍只在各自浏览器里保存数据。
- 若通过网站分享，腾讯地图 Key 需配置**允许该域名**（见下一节）。

## 四、腾讯地图 Key（正式分享前必读）

当前页面内嵌的是**有有效期的临时 Key**，适合开发测试，**不适合长期对外分享**。

### 正式分享前请完成

1. 前往 [腾讯位置服务](https://lbs.qq.com/) 申请 **JavaScript API GL** 的长期 Key。
2. 在控制台开启所需能力：
   - **JSAPI**（地图显示、标记、路线等）
   - **WebService / service**（地点搜索建议）
3. 若使用静态网站分享，在 Key 的**域名白名单**中加入你的托管域名（如 `你的用户名.github.io`）。
4. 将新 Key 交给**维护页面的人**（通常是有 Python 环境的开发者），在 `generate_html.py` 文件**顶部**找到 `KEY = "..."` 一行，替换为新 Key，然后运行：

   ```
   C:/Python314/python.exe generate_html.py
   ```

5. 重新生成后，用新的 `itinerary.html` 进行分享。

> **安全提示：** Key 会出现在浏览器页面源码中，任何人可见。这是前端地图的正常做法，**不要把 Key 当作服务器密码**；请用域名白名单和配额限制来管控滥用。

**普通接收者无需安装 Python**，只要收到已更新 Key 的 `itinerary.html` 即可使用。

## 六、常见问题

### 地图空白

可能原因与处理：

1. **未联网** — 连接网络后刷新页面。
2. **Key 已过期** — 联系分享者更新 Key 并重新生成 `itinerary.html`。
3. **域名未授权** — 若通过网站打开，需在腾讯控制台把该域名加入 Key 白名单。
4. **浏览器拦截脚本** — 尝试换用 Chrome / Edge，并允许加载外部脚本。

### 修改消失了

1. 若有 JSON 备份：点击「导入 JSON」，选择最近导出的文件。
2. 若无备份：只能点击「恢复初始」回到默认行程，之前的修改无法找回。

### 地点搜索不可用

1. 检查 Key 是否开启 **WebService / service** 权限。
2. 若搜索仍失败，可改用 **地图点击选点**：在地图上单击目标位置，坐标会自动填入表单。

### 无法拖拽排序

- **桌面浏览器**：请用鼠标拖动时间轴上的行程卡片；请确认未在移动端浏览器中使用。
- **手机 / 平板**：按设计不提供拖拽，可在桌面电脑上调整顺序后再导出 JSON 分享。

### 导入 JSON 后没有变化

- 确认选择的是正确的 `.json` 文件（导出时的 `shandong-itinerary-YYYY-MM-DD.json`）。
- 若 JSON 文件损坏或格式不对，页面会提示错误，**不会**覆盖当前行程。

## 七、验收清单（收到最终文件后请逐项勾选）

请在**联网**的**桌面浏览器**中打开 `itinerary.html`，逐项测试并勾选：

- [ ] **DAY 切换：** 依次点击 DAY 1、DAY 2、DAY 3，地图与时间轴均切换到对应日期，标记带编号，路线带箭头。
- [ ] **地图缩放：** 放大、缩小地图时，标记位置固定、不聚合为簇。
- [ ] **地图选点新增：** 在表单中开启地图选点，单击地图填入坐标，提交后新点出现在列表和地图上；刷新页面后仍存在。
- [ ] **搜索新增：** 在搜索框输入地名，选择建议项，提交后新增成功；刷新后仍存在。
- [ ] **拖拽排序：** 拖动某天的行程卡片改变顺序，刷新页面后顺序保持不变。
- [ ] **导出 → 恢复初始 → 导入：** 先导出 JSON；点击「恢复初始」回到默认行程；再导入刚才的 JSON，行程完整恢复。
- [ ] **损坏 JSON：** 用文本编辑器故意破坏 JSON 内容后尝试导入，页面应报错且**不改变**当前行程。

若某项无法完成，请记录浏览器名称、是否本地文件或网站打开，以及具体现象，反馈给分享者。
