# 纯 HTML 行程编辑器设计

## 目标

将现有旅行行程页增强为无需后端、无需数据库的单文件应用。使用者通过静态托管 URL 或本地 HTTP 服务打开页面，即可：

- 按每日当前顺序查看带数字的地图标记。
- 查看带方向箭头的完整路线。
- 切换 DAY 标签并自动显示当天全部点位和路线。
- 通过地图点击或腾讯地点搜索新增点。
- 拖拽当天卡片调整顺序。
- 在当前浏览器中自动保存修改。
- 导出和导入 JSON，用于备份、迁移设备和分享修改后的行程。

## 交付形式

运行 `generate_html.py` 后生成一个自包含的 `itinerary.html`。行程基础数据、样式和业务 JavaScript 全部内嵌；仅腾讯地图脚本需要联网加载。

普通使用者不需要 `generate_html.py`、`get_pois.py` 或 `coords.json`。推荐将 `itinerary.html` 重命名为 `index.html` 后上传静态托管，并分享 URL；如需分享修改后的数据，再分享导出的 JSON 文件。

腾讯 JSAPI GL 不支持 `file://`。本地测试需在项目目录运行 `C:/Python314/python.exe -m http.server 8000`，并访问 `http://localhost:8000/itinerary.html`。直接双击时地图、搜索和地图选点不可保证可用；时间轴、导入导出、恢复及部分本地保存能力可能仍可用。

## 数据状态

页面维护三层数据：

1. `BASE_POIS`：生成 HTML 时内嵌的基础行程，不在浏览器中直接修改。
2. `workingState`：当前页面实际展示和编辑的完整行程。
3. `localStorage`：自动保存的完整行程快照。

每个点拥有稳定的非空字符串 ID；校验时去除首尾空白，拒绝数字、对象和数组 ID。基础点使用可重复生成的 ID，自定义点使用浏览器生成的 UUID。每个点包含：

- `id`
- `day`
- `position`
- `time`
- `title`
- `name`
- `desc`
- `city`
- `cat`
- `lat`
- `lng`
- `source`

加载优先级为：有效的 `localStorage` 快照优先，否则使用 `BASE_POIS`。

## 本地保存

使用版本化键名 `itinerary-editor-state-v1` 保存：

- 数据格式版本。
- 保存时间。
- 全部行程点。

新增或拖拽排序后立即写入。保存成功后显示简短状态；写入失败时明确提示用户立即导出 JSON。

`localStorage` 只属于当前浏览器和当前网页来源。清理浏览器数据、无痕模式、换浏览器、换设备或改变文件访问方式时，数据可能不可用，因此 JSON 导出是跨设备和长期备份机制。

## 地图和每日路线

点击 DAY 标签后始终重建时间轴；仅在腾讯地图脚本和地图实例可用时执行地图渲染：

1. 取当天点位并按 `position` 排序。
2. 重建左侧卡片和从 1 开始的编号。
3. 为每个点生成带数字及类别颜色的固定像素 SVG 标记。
4. 清空旧标记并绘制当天全部标记。
5. 按编号顺序绘制 `MultiPolyline`。
6. 折线设置 `showArrow: true` 和箭头间距。
7. 用 `fitBounds` 显示当天全部点。

标记不聚合、不因缩放主动隐藏。地图缩得很小时，距离接近的标记允许相互遮挡。

## 新增点

页面提供“添加行程点”面板，默认选择当前 DAY。支持：

- 地图选点：进入选点模式，点击地图取得经纬度。
- 地点搜索：使用腾讯地图 JSAPI GL `service` 附加库搜索关键词和城市；每次搜索前对 Suggestion 服务调用 `setRegion(city)`，再以仅含关键词的参数请求建议，并保留 8 条分页上限。

日期、时间、标题、类别和坐标必填。新增点最初按时间插入当天位置；相同时间排在已有同时间点之后。保存后立即重排编号和路线。

地点搜索不可用时显示错误，但地图点击选点仍可使用。

## 拖拽排序

当天卡片支持原生拖拽。放置完成后：

- 根据卡片顺序重写当天 `position`。
- 立即更新编号、标记和箭头路线。
- 自动写入 `localStorage`。

拖拽后以用户顺序为准，不再强制按时间排序。之后新增的点仍按时间确定首次插入位置。

## JSON 导出与导入

### 导出

点击“导出 JSON”下载一个 UTF-8 文件，文件名包含导出日期。内容包含：

- `schemaVersion`
- `exportedAt`
- `title`
- 完整 `points` 数组

导出的文件代表当前页面全部行程，可用于备份或分享。

### 导入

点击“导入 JSON”选择文件。页面先完成结构校验：

- JSON 可解析。
- 格式版本受支持。
- 点 ID 唯一。
- 日期、时间、类别和坐标有效。
- 每天顺序可规范化。

校验通过后，弹出确认提示，说明导入会替换当前行程。用户确认后才覆盖 `workingState` 和 `localStorage`，随后重画当前 DAY。

解析或校验失败时不修改任何现有数据。

## 恢复基础行程

提供“恢复初始行程”操作。二次确认后删除对应 `localStorage` 快照并恢复内嵌 `BASE_POIS`。该操作不会删除用户已经下载的 JSON 备份。

## 分享方式

### 分享初始行程

将单个 `itinerary.html` 重命名为 `index.html` 后上传静态托管，并分享访问 URL。对方修改后保存在其自己的浏览器中。

### 分享修改后的行程

发送：

- 相同版本页面的静态托管 URL
- 从页面导出的 JSON 文件

对方打开 URL 后导入 JSON，即可得到分享者修改后的完整行程。

### 静态网站分享

将 `itinerary.html` 作为 `index.html` 上传至静态托管服务是推荐分享方式。访问者仍各自拥有独立的浏览器数据，互不自动同步；需要通过导出/导入 JSON 交换修改。

## 腾讯地图 Key

腾讯地图 JSAPI Key 会出现在浏览器代码中，不能视为服务器秘密。用于正式分享前需要：

- 使用有效的长期 Key，不能依赖即将过期的临时 Key。
- 加载脚本时加入 `libraries=service`。
- 静态托管时在腾讯位置服务控制台配置允许访问的域名。

## 错误处理

- `file://` 访问：明确提示腾讯 JSAPI GL 不支持直接双击，并给出本地 HTTP 地址。
- 地图脚本、Key 或初始化失败：仍选择首日，保留 DAY 标签、时间轴、新增表单、自动保存、导入、导出和恢复；地图选点禁用或提示不可用。
- DAY 切换：始终渲染时间轴；只有地图和 `TMap` 可用时才渲染地图，地图渲染函数自身也执行可用性检查。
- `localStorage` 失败：保留内存数据并提示导出。
- 地点搜索失败：允许改用地图选点。
- JSON 导入失败：保持原数据完全不变并指出错误。
- 空日程或单点日程：不绘制无意义的折线。

## 验收

- DAY 切换后地图只显示当天全部点和按顺序连接的箭头路线。
- 数字标记在缩放时保持固定尺寸且不被聚合。
- 地图点击和地点搜索均可新增点。
- 新增与拖拽排序后刷新页面，修改仍存在。
- 导出后恢复初始行程，再导入文件，可以完整恢复修改。
- 无效 JSON 不会破坏现有行程。
- 将 HTML 和 JSON 交给另一浏览器后，可通过导入重现相同行程。
