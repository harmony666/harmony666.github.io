> **现役说明（2026-07-22）：** 本文是编辑器 UI/数据模型基线，但下列实现已与初稿不同，以代码为准：
> - 排序改为**按时间**，**无拖拽排序**；卡片提供「编辑」。
> - 路线优先用百度 `BMap.DrivingRoute` 逐段驾车折线，失败回退直线。
> - 手机端「行程 / 地图」分段切换；地图容器不可 `display:none`。
> - 生成物仅写 `index.html`；**线上入口** http://124.222.108.66/（腾讯云轻量 Nginx；共享同步见 `2026-07-22-cloud-shared-itinerary-design.md`）。
> - 卡片支持编辑与**删除**（删除需确认）；地图为百度 JS API（坐标存储 GCJ-02，展示转 BD-09）。
> - 现役行程与预算以 `coords.json` / 生成后的 `BASE_POIS` 为准（约 ¥5745.65）。

## 目标

将旅行行程页做成可在浏览器中编辑的单页应用：使用者打开托管 URL 或本地 HTTP 服务即可查看与编辑。可选配置 `API_BASE` 后，与同机 Node API 做**一份共享行程**同步（详见云端规格）；未配置时仍为纯本机 `localStorage`。

页面能力包括：

- 按每日当前顺序查看带数字的地图标记。
- 查看带方向箭头的完整路线（现役为驾车路网折线，失败时直线）。
- 切换 DAY 标签并自动显示当天全部点位和路线。
- 通过地图点击或百度地点搜索新增点。
- 编辑已有点；删除需二次确认；当天按时间自动排序（初稿中的拖拽排序未采用）。
- 在当前浏览器中缓存修改；配置云端时写共享文档。
- 导出和导入 JSON，用于备份与迁移。

## 交付形式

运行 `generate_html.py` 后生成一个自包含的 **`index.html`**。行程基础数据、样式和业务 JavaScript 全部内嵌；仅百度地图脚本需要联网加载。`API_BASE` 在生成时注入（线上为 `http://124.222.108.66`；留空则纯本地模式）。

普通使用者不需要 `generate_html.py` 或 `coords.json`。打开 http://124.222.108.66/（或仓库中的 `index.html` 经本地 HTTP 打开）即可使用。

百度地图 JS API 不支持 `file://`。本地测试需在项目目录运行 `C:/Python314/python.exe -m http.server 8000`，并访问 `http://localhost:8000/`。直接双击时地图、搜索和地图选点不可保证可用；时间轴、导入导出、恢复及部分本地保存能力可能仍可用。

部署：`python scripts/deploy_vps.py` 将 `index.html` 放到服务器 `/var/www/itinerary/`，并由 Nginx 提供。

## 数据状态

页面维护这些数据层：

1. `BASE_POIS`：生成 HTML 时内嵌的基础行程，不在浏览器中直接修改。
2. `workingState`：当前页面实际展示和编辑的完整行程。
3. `localStorage`（键 `itinerary-editor-state-v1`）：成功拉取/保存后的本地缓存；云端不可用时只读兜底。
4. （可选）云端文档：配置 `API_BASE` 时，以服务器 JSON 为准，约 5 秒轮询。

每个点拥有稳定的非空字符串 ID；校验时去除首尾空白，拒绝数字、对象和数组 ID。基础点使用可重复生成的 ID，自定义点使用浏览器生成的 UUID。每个点包含：

- `id`
- `day`
- `position`
- `time`
- `title`
- `name`
- `desc`
- `city`
- `cat`
- `lat`
- `lng`
- `source`

加载优先级（现役）：配置了云端时优先拉共享文档并写入缓存；云端失败则用有效 `localStorage`；再否则用 `BASE_POIS`。纯本地模式：有效 `localStorage` 优先，否则 `BASE_POIS`。

## 本地保存与云端

未配置 `API_BASE` 时：新增或编辑后立即写入 `localStorage`；失败时提示导出 JSON。

配置了 `API_BASE` 时：写操作需共享口令（`sessionStorage` + 请求头 `X-Edit-Password`）；成功后更新云端 `version` 并刷新本地缓存。`localStorage` 只属于当前浏览器来源，不能替代云端共享。

## 地图和每日路线

点击 DAY 标签后始终重建时间轴；仅在百度地图脚本和地图实例可用时执行地图渲染：

1. 取当天点位并按 `position` 排序。
2. 重建左侧卡片和从 1 开始的编号。
3. 为每个点生成带数字及类别颜色的固定像素 SVG 标记。
4. 清空旧标记并绘制当天全部标记。
5. 按编号顺序绘制驾车折线（失败回退直线 `MultiPolyline`）。
6. 折线设置 `showArrow: true` 和箭头间距。
7. 用 `fitBounds` 显示当天全部点。

标记不聚合、不因缩放主动隐藏。地图缩得很小时，距离接近的标记允许相互遮挡。

## 新增点

页面提供“添加行程点”面板，默认选择当前 DAY。支持：

- 地图选点：进入选点模式，点击地图取得经纬度。
- 地点搜索：使用百度 `BMap.LocalSearch`（可按城市限定）；结果坐标从 BD-09 转回 GCJ-02 写入表单，最多 8 条。

日期、时间、标题、类别和坐标必填。新增点最初按时间插入当天位置；相同时间排在已有同时间点之后。保存后立即重排编号和路线。

地点搜索不可用时显示错误，但地图点击选点仍可使用。

## 排序（现役：按时间）

> 初稿曾规划原生拖拽排序；实现后改为时间排序，避免与「按时刻行程」冲突。

- 新增与编辑保存后，当天按 `time` 升序重算 `position`。
- 时间轴编号与地图路线顺序一致。
- `ItineraryCore.reorderDay` 仍保留在核心库中，但 UI 不再暴露拖拽。

## JSON 导出与导入

### 导出

点击“导出 JSON”下载一个 UTF-8 文件，文件名包含导出日期。内容包含：

- `schemaVersion`
- `exportedAt`
- `title`
- 完整 `points` 数组

导出的文件代表当前页面全部行程，可用于备份或分享。

### 导入

点击“导入 JSON”选择文件。页面先完成结构校验：

- JSON 可解析。
- 格式版本受支持。
- 点 ID 唯一。
- 日期、时间、类别和坐标有效。
- 每天顺序可规范化。

校验通过后，弹出确认提示，说明导入会替换当前行程。用户确认后才覆盖 `workingState`（及本地缓存）；若已配置云端，写云端需口令。随后重画当前 DAY。

解析或校验失败时不修改任何现有数据。

## 恢复基础行程

提供“恢复初始行程”操作。二次确认后：

- 未配置云端：删除对应 `localStorage` 并恢复内嵌 `BASE_POIS`。
- 已配置云端：口令通过后重置**云端**文档为 seed（全员生效），并刷新本地缓存。

该操作不会删除用户已经下载的 JSON 备份。

## 分享方式

### 推荐：线上共享站点

分享 http://124.222.108.66/ 。只读无需口令；编辑需共享口令，保存后其他人刷新或等待轮询即可看到。细节见 `2026-07-22-cloud-shared-itinerary-design.md`。

### 纯静态 / 本机

未配置 `API_BASE` 时，访问者各自拥有独立的浏览器数据，互不自动同步；需通过导出/导入 JSON 交换修改。

### 分享修改后的行程（离线备份）

发送页面 URL（或本机 `index.html` 的访问方式）以及从页面导出的 JSON；对方导入即可得到相同行程快照。

## 百度地图 Key

百度地图 AK 会出现在浏览器代码中，不能视为服务器秘密。用于正式分享前需要：

- 使用有效的浏览器端长期 AK。
- 在控制台配置 Referer 白名单（现役含 `124.222.108.66`、`localhost:8000`）。
- 开通 JavaScript API、地点检索、驾车路线等相关服务。

## 错误处理

- `file://` 访问：明确提示百度地图不支持直接双击，并给出本地 HTTP 地址（`http://localhost:8000/`）。
- 地图脚本、Key 或初始化失败：仍选择首日，保留 DAY 标签、时间轴、新增表单、自动保存、导入、导出和恢复；地图选点禁用或提示不可用。
- DAY 切换：始终渲染时间轴；只有地图和 `BMap` 可用时才渲染地图，地图渲染函数自身也执行可用性检查。
- `localStorage` 失败：保留内存数据并提示导出。
- 云端失败：提示并尽量使用本地缓存（只读兜底）。
- 地点搜索失败：允许改用地图选点。
- JSON 导入失败：保持原数据完全不变并指出错误。
- 空日程或单点日程：不绘制无意义的折线。

## 验收

- DAY 切换后地图只显示当天全部点和按顺序连接的箭头路线。
- 数字标记在缩放时保持固定尺寸且不被聚合。
- 地图点击和地点搜索均可新增点。
- 新增或编辑后刷新页面，修改仍存在（本机或云端配置下）。
- 切换 DAY 时地图拟合当天点位；手机端可在行程/地图间切换。
- 驾车路线计算失败时退回直线且页面可用。
- 导出后恢复初始行程，再导入文件，可以完整恢复修改。
- 无效 JSON 不会破坏现有行程。
- 配置云端时，两台设备在数秒内可见同一份行程；生成物仅为 `index.html`。
