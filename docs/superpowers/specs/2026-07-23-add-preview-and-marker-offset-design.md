# 添加预览 + 同坐标编号错位（设计）

> 日期：2026-07-23  
> 状态：**已落地**（grilling 收口后已实现并部署）  
> 范围：编辑器 UI / `ItineraryCore` / `generate_html.py`；云端 API 契约不变

## 问题

1. 添加/编辑时右侧（或手机全屏）表单挡住地图，看不到选点与插入位置，易造成回头路。  
2. 当天首尾等同坐标点叠在一起时，只看得见最后一个编号。

## 决策

| 项 | 选择 |
|----|------|
| 目标 | 空间位置 + 时间顺序都要可见；先做「看得见」 |
| 桌面表单 | 固定在**左侧**，右侧地图保持可见且可点选 |
| 手机表单 | **底部半屏抽屉**，上方留地图（打开编辑时切到地图视图） |
| 预览钉 | 有有效坐标即显示临时钉；保存前不写入正式行程 |
| 预览线 | 有有效时间后，按当天时间序画虚线「前一点 → 新点 → 后一点」 |
| 不猜时间 | 不按地理位置自动建议插入时间 |
| 叠点 | 当天近似同坐标的点：**钉锚在真坐标**，**编号图标轻微像素错位** |
| 取消 | 清除全部预览叠加层 |

## 核心逻辑（可测）

- `computeMarkerPixelOffsets(points, threshold?)`：按近似同坐标分簇，簇内给出 `{id, offsetX, offsetY}`（像素）。  
- `findTimeNeighbors(dayPoints, draft, excludeId?)`：返回 `{ prev, next }`，供虚线与可选文案「插在 A 与 B 之间」。

## UI 约束

- 正式路线（实线驾车）仍只基于已保存点；预览虚线为独立叠加。  
- 编辑已有点时：邻居计算排除自身；正式钉可仍显示，预览钉表示「将要变成的位置」。  
- 遮罩不得挡住地图点击（桌面左侧面板 `pointer-events` 隔离）。
